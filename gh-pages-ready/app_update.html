Kontrola verze mobilní aplikace

Aplikace CzechMate si při otevření nastavení stáhne soubor app_update.json z kořene tohoto webu a porovná semver verzi z pubspec.yaml s polem latest_version.

Zdroj pravdy pro klienta je vždy JSON (strojově čitelný). Tato stránka popisuje schéma pro údržbu.

Umístění

app_update.json (stejný obsah jako v repu gh-pages-ready/app_update.json).

Schéma

  • latest_version (řetězec, povinné) — nejnovější verze aplikace. Pokud je instalovaná verze starší, klient zobrazí upozornění.
  • min_supported_version (řetězec, volitelné) — pod touto verzí je doporučená okamžitá aktualizace a výraznější zobrazení.
  • release_page_url (řetězec URL, volitelné) — kam má vést tlačítko ke stažení. Pokud chybí, použije se výchozí produktová stránka z kódu aplikace.

Příklad

{
  "latest_version": "1.7.4",
  "min_supported_version": "1.6.0",
  "release_page_url": "https://alfredkrutina.github.io/chess_esp32_c6_devkit/downloads.html"
}

Po vydání nové aplikace zvedni latest_version (a případně min_supported_version) v souladu s flutter_czechmate/pubspec.yaml, commitni změnu a nech proběhnout deploy GitHub Pages.
